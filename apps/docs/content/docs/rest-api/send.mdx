The `send` endpoint is the main way to send emails via the Rest API. To use it you need to pass in an API token in the `Authorization` header and a JSON body with the details of the email to send.

## Example usage

<Tabs items={['cURL', 'Javascript']}>
  <Tab value="cURL">
    ```bash
    curl --request POST \
    --url https://api.volamail.com/send \
    --header 'Authorization: Bearer vl_my-volamail-token' \
    --header 'Content-Type: application/json' \
    --data '{
      "template": "my-welcome-email",
      "from": "noreply@company.com",
      "to": "john.doe@example.com",
      "data": {
        "user_name": "Mauro"
      }
    }'
    ```
  </Tab>
  <Tab value="Javascript">
    ```js
    fetch("https://api.volamail.com/send", {
      method: "POST",
      headers: {
        "Authorization": `Bearer ${process.env.VOLAMAIL_API_TOKEN}`,
        "Content-Type": "application/json"
      },
      body: JSON.stringify({
        template: "my-welcome-email",
        from: "noreply@company.com",
        to: "john.doe@example.com",
        data: {
          user_name: "Mauro"
        }
      })
    })
    ```
  </Tab>
</Tabs>

## Reference

### Headers

<Property name="Authorization" type="Bearer <token>" required />

### JSON body

<Property name="template" type="string" required>
  The ID of the template to send.
</Property>
<Property name="from" type="string (email)" required>
  The email address to send the email from. This needs to be from a verified domain.
</Property>
<Property name="to" type="string (email)" required>
  The email address to send the email to.
</Property>
<Property name="data" type="Record<string, string>">
  The record of variables to pass to the template.
</Property>
